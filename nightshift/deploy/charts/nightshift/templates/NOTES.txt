  kubectl -n {{ .Release.Namespace }} exec openbao-0 -c openbao -- \
    env BAO_TOKEN="$(...login as admin...)" \
    bao write auth/userpass/users/<new-user> password=<new-pass>

To rotate the seeded admin password: delete the Secret
{{ include "nightshift.openbaoAdminSeedSecret" . }} + re-run the
post-install bootstrap Job (helm upgrade triggers it automatically).

{{- if and .Values.ui.enabled (not .Values.ui.ingress.host) }}

Browser login from a kubectl port-forward (no Ingress) requires a few
extra steps because OpenBao's OIDC issuer URL is the in-cluster service
DNS, which doesn't resolve from your laptop. Pick one:

  A. Quick: add a hosts-file entry mapping the OpenBao service name
     to localhost, then port-forward openbao on 8200:
       sudo sh -c 'echo "127.0.0.1 openbao.{{ .Release.Namespace }}.svc" >> /etc/hosts'
       kubectl -n {{ .Release.Namespace }} port-forward svc/openbao 8200:8200
     Browser will reach the OIDC authorize URL at the same hostname
     the in-cluster API verifies against.

  B. Production: install with a real ingress + DNS (set ui.ingress.host
     and openbao.oidc.issuerURL to externally-resolvable URLs). Both
     the API and the browser will use the same public hostname.

For both, also set:
  --set ui.publicURL=https://<your-ui-host>     # e.g. http://localhost:13000 for dev
  --set 'ui.trustedOrigins={<extra-origins>}'   # if any differ from publicURL

{{- end }}
