# services/python-api/

FastAPI service with Prometheus metrics — the canonical Python sample for the DevOps pipeline.

## What's Here

```
main.py            # FastAPI app — /health, /ready, /metrics, /items endpoints
requirements.txt   # Runtime deps: fastapi, uvicorn, prometheus-client
pyproject.toml     # Project metadata
Dockerfile         # Multi-stage distroless image (mirrors templates/dockerfiles/python/)
helm/              # Helm chart (generated by tools/helm_gen.py)
terraform/         # AWS Terraform (generated by tools/terraform_gen.py)
secrets/           # ESO manifests (generated by tools/secrets_manager.py)
tests/             # pytest test suite
```

## How to Use / Run

```bash
# Install deps
pip install -r requirements.txt

# Run locally on port 8000
uvicorn main:app --reload --port 8000

# Health check
curl http://localhost:8000/health

# Prometheus metrics
curl http://localhost:8000/metrics

# Run tests
pytest tests/ -v

# Build Docker image
docker build -t python-api:local .

# Run container
docker run -p 8000:8000 python-api:local
```

## Key Details

- **Port**: 8000
- **Health endpoint**: `GET /health` → `{"status": "ok", "service": "python-api", "uptime": <seconds>}`
- **Metrics endpoint**: `GET /metrics` — Prometheus text format
- Prometheus metrics exported: `http_requests_total` (Counter), `http_request_duration_seconds` (Histogram), `http_active_requests` (Gauge)
- Metrics are tracked via middleware — every route is automatically instrumented
- Dockerfile uses `gcr.io/distroless/python3-debian12:nonroot` as final stage (~60 MB image)
- Runs as UID 65532 (nonroot) — no shell in the final image
