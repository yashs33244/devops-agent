# services/go-api/

Go + Chi router service with Prometheus metrics — the canonical Go sample for the DevOps pipeline.

## What's Here

```
main.go            # Chi router — /health, /ready, /metrics, /items endpoints
go.mod / go.sum    # Module: chi/v5, prometheus/client_golang
Dockerfile         # Multi-stage distroless image (mirrors templates/dockerfiles/go/)
helm/              # Helm chart (generated by tools/helm_gen.py)
terraform/         # AWS Terraform (generated by tools/terraform_gen.py)
secrets/           # ESO manifests (generated by tools/secrets_manager.py)
main_test.go       # Go test suite
go-api/            # (sub-package if present)
```

## How to Use / Run

```bash
# Run locally on port 8080
go run main.go

# Health check
curl http://localhost:8080/health

# Prometheus metrics
curl http://localhost:8080/metrics

# Run tests
go test ./...

# Build binary
go build -o go-api .

# Build Docker image
docker build -t go-api:local .

# Run container
docker run -p 8080:8080 go-api:local
```

## Key Details

- **Port**: 8080 (set via `PORT` env var; defaults to 8080)
- **Health endpoint**: `GET /health` → JSON with status and uptime
- **Metrics endpoint**: `GET /metrics` — Prometheus text format (promhttp handler)
- Prometheus metrics: `http_requests_total` (CounterVec by method/path/status), `http_request_duration_seconds` (HistogramVec by path)
- Final Docker image uses `gcr.io/distroless/static-debian12:nonroot` (~5–15 MB, no shell)
- Runs as UID 65532 (nonroot)
- Chi middleware adds request logging and panic recovery
