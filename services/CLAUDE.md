# services/

Three sample services used to demonstrate and test the DevOps agent pipeline end-to-end.

## What's Here

| Directory | Language | Port | Framework |
|-----------|----------|------|-----------|
| `python-api/` | Python 3.12 | 8000 | FastAPI + Prometheus metrics |
| `go-api/` | Go 1.21+ | 8080 | Chi router + Prometheus metrics |
| `nextjs-demo/` | Node.js | 3000 | Next.js (see AGENTS.md for version caveats) |

Each service contains:
- `Dockerfile` — production multi-stage image
- `helm/` — Helm chart for Kubernetes deployment
- `terraform/` — cloud infrastructure (pre-generated by `tools/terraform_gen.py`)
- `secrets/` — ESO manifests (pre-generated by `tools/secrets_manager.py`)

## How to Use / Run

```bash
# Run python-api locally
cd python-api && pip install -r requirements.txt
uvicorn main:app --port 8000

# Run go-api locally
cd go-api && go run main.go

# Run nextjs-demo locally
cd nextjs-demo && npm install && npm run dev

# Run tests for python-api
cd python-api && pytest tests/

# Run tests for go-api
cd go-api && go test ./...

# Lint Helm chart
helm lint python-api/helm

# Full pipeline test (from repo root)
python3 tools/test_runner.py \
  --service python-api \
  --repo-path services/python-api \
  --terraform-dir services/python-api/terraform \
  --helm-dir services/python-api/helm \
  --cloud aws
```

## Key Details

- All three services expose `/health` and `/metrics` endpoints — required by Helm chart probes
- Prometheus metrics follow the same naming convention across all services: `http_requests_total`, `http_request_duration_seconds`
- Grafana dashboard (`templates/monitoring/grafana/`) is pre-wired to scrape these metric names
- `terraform/` and `secrets/` directories are **generated output** — regenerate with `tools/terraform_gen.py` and `tools/secrets_manager.py` rather than editing by hand
- The services are intentionally minimal — they exist to validate the tooling pipeline, not as production templates
