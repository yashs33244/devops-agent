# agents/

Index of all five embedded sub-agent repositories used for reference and advanced use cases.

## What's Here

| Directory | Language | Purpose |
|-----------|----------|---------|
| `holmesgpt/` | Python (Poetry) | AI-powered incident investigation — queries K8s, Prometheus, Grafana to diagnose alerts |
| `kagent/` | Go + Python + Next.js | Kubernetes-native AI agent framework — CRDs, controllers, agent runtime |
| `nightshift/` | Go (multi-binary) | Cost-optimization scheduler — scale-to-zero orchestration via gRPC API + worker protocol |
| `opensre/` | Python (uv) | SRE runbook automation — LLM-driven root-cause analysis pipeline |
| `plural/` | Elixir (Mix/Phoenix) | GitOps multi-cloud deployment platform — Helm, ArgoCD, multi-tenant control plane |

## How They Interact with the Main Repo

- `tools/cost_optimize.py` complements `nightshift/` — nightshift handles scheduled scale-down; cost_optimize applies KEDA manifests immediately.
- `tools/test_runner.py` can invoke holmesgpt investigations against a running cluster to validate deployed services.
- `plural/` is the GitOps back-end reference for the CD pipeline generated by `tools/cicd_setup.py`.
- `kagent/` demonstrates how to run AI agents on Kubernetes — patterns reused in Helm templates under `templates/helm/`.

## Key Details

- These are **read-only reference implementations** for v1 of this monorepo. Do not modify them unless explicitly working on sub-agent development.
- Each sub-directory has its own dependency manager (Poetry, go.mod, uv, Mix). Do not mix them.
- Sub-agents each have their own `CLAUDE.md` or `AGENTS.md` — open those before working inside a sub-agent directory.

## Related
